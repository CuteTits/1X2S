rocket_launch Quickstart

Send a JSON body to the search endpoint and include your API key as a bearer token.

Method POST
Endpoint https://api.1ex.io/v1/search
Auth Header Authorization: Bearer YOUR_API_KEY
Content Type application/json
Example Request 
curl -X POST "https://api.1ex.io/v1/search" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "market sentiment around clean energy stocks",
    "location": "United States",
    "max_results": 8,
    "include_summary": true
  }'

terminal Multi-language Examples

Use the same endpoint and payload across languages. Replace YOUR_API_KEY before running.

Python
import requests

url = "https://api.1ex.io/v1/search"

    "Authorization": "Bearer YOUR_API_KEY",
    "Content-Type": "application/json"

    "query": "market sentiment around clean energy stocks",
    "location": "United States",
    "max_results": 8,
    "include_summary": True
}

response = requests.post(url, headers=headers, json=payload, timeout=30)
print(response.json())

Go
package main

import (
  "bytes"
  "fmt"
  "io"
  "net/http"
)

func main() {
  body := []byte(`{"query":"market sentiment around clean energy stocks","location":"United States","max_results":8,"include_summary":true}`)
  req, _ := http.NewRequest("POST", "https://api.1ex.io/v1/search", bytes.NewBuffer(body))
  req.Header.Set("Authorization", "Bearer YOUR_API_KEY")
  req.Header.Set("Content-Type", "application/json")

  resp, _ := http.DefaultClient.Do(req)
  defer resp.Body.Close()
  out, _ := io.ReadAll(resp.Body)
  fmt.Println(string(out))
}
C++ (libcurl)
#include <curl/curl.h>

int main() {
  CURL* curl = curl_easy_init();
  if (!curl) return 1;

  const char* json = "{\"query\":\"market sentiment around clean energy stocks\",\"location\":\"United States\",\"max_results\":8,\"include_summary\":true}";
  struct curl_slist* headers = NULL;
  headers = curl_slist_append(headers, "Authorization: Bearer YOUR_API_KEY");
  headers = curl_slist_append(headers, "Content-Type: application/json");

  curl_easy_setopt(curl, CURLOPT_URL, "https://api.1ex.io/v1/search");
  curl_easy_setopt(curl, CURLOPT_POST, 1L);
  curl_easy_setopt(curl, CURLOPT_POSTFIELDS, json);
  curl_easy_setopt(curl, CURLOPT_HTTPHEADER, headers);
  curl_easy_perform(curl);

  curl_slist_free_all(headers);
  curl_easy_cleanup(curl);
  return 0;
}

verified Best Practices

Keep queries intent-focused and response handling deterministic for stable automation behavior.

Be Specific with Query Intent

Explicit entities and goals improve ranking precision.

Tune Result Limits by Use Case

Lower limits for low-latency agents, higher for analyst review.

Cache by Query Fingerprint

Reduce duplicate calls while keeping data fresh.

warning Error Handling

Use status codes plus error payloads to drive retries and user-facing fallback behavior.

StatusError TypeRecommended Action
400invalid_requestValidate payload fields.
401authentication_errorCheck bearer token and key state.
429rate_limitedApply retry and exponential backoff.
500server_errorRetry safely, then contact support.
