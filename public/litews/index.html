Overview

LiteWS is an AI-powered web scraping and data extraction tool designed to streamline the process of gathering and analyzing online information. Built with Python, LiteWS integrates advanced AI models and NLPs to enhance its scraping capabilities, allowing users to extract structured data from a variety of web sources efficiently. The tool supports multiple AI providers, enabling users to leverage state-of-the-art language models for data summarization and analysis. LiteWS is ideal for researchers, analysts, and developers looking to automate data collection tasks while ensuring high-quality outputs through AI-driven insights.

Getting Started

Installation

Prerequisites

Ensure you have Python 3.12 or higher installed. You can download it from python.org Download the LiteWS package from Github System Requirements: >8GB RAM, >500MB Available Storage

Install Required Libraries

Within the file package there is a requirements.txt file which contains all the required libraries as a package. Run the following string in the terminal to download the requirements package with all the libraries. 
            pip install requirements.txt -r
Or optionally, use pip to install the necessary libraries manually: 
            pip install aiohttp playwright requests beautifulsoup4 pydantic google-genai python-dotenv ddgs

Playwright Setup

After installing Playwright, run the following command to install the necessary browser binaries: 
        playwright install

API Integration

Chosing AI/API Model

To get structured responses and summaries from the web results you can integrate an API-based AI. You’ll interact with it through Python SDKs already integrated in the code. Specific usage depends on what AI platform you choose. error Keep in mind that using external APIs may incur fees, such as pay-as-you-go charges. 1X2S does not include or cover any costs associated with API usage. Fees and plans are usually stated by respective API provider. LiteWS supports by default the following APIs:
  • OpenAI
  • Google Cloud AI
  • Mistral AI
  • DeepSeek API
  • Hugging Face Inference API
  • Grok API
The code can be modified to support other AI models.

Accessing an API Key

Every API requires a unique API key, which is necessary to make API calls. You can obtain these keys directly from the corresponding API or AI service provider.
  • OpenAI API
  • Google AI
  • Mistral API
  • DeepSeek API
  • Hugging Face APi
  • xAI Grok API

Setting Up Environment Variables

Your API keys should be securely stored in an environment variable file located in your project directory. If configured correctly, the script will automatically retrieve the necessary API keys from the .env file. To begin, create a file named .env in the root directory of your project. Next, choose the API provider you intend to use from the list below. Copy the corresponding line, paste it into your .env file, and replace PLACEHOLDER with the actual API key provided by that service. 
OPENAI_API_KEY=PLACEHOLDER
        
GEMINI_API_KEY=PLACEHOLDER
        
MISTRAL_API_KEY=PLACEHOLDER
        
DEEPSEEK_API_KEY=PLACEHOLDER
        
HFI_API_KEY=PLACEHOLDER
        
XAI_API_KEY=PLACEHOLDER
error Never share your .env file or API keys with anyone. These credentials are private and should be kept secure at all times.
